Git Tutorial

What is Git?

Git is a distributed version-control system. It snapshots your project history so you can time-travel, collaborate, and experiment safely.

Install Git

  • Windows: winget install Git.Git or download from git-scm.com
  • macOS: brew install git
  • Debian/Ubuntu: sudo apt install git

Verify Install

git --version

Global Config (once per machine)

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main
git config --global core.autocrlf input # mac/Linux
git config --global core.autocrlf true # Windows

Get Help Instantly

git help <command>
git <command> -h
git config --list

everyday 5-command Workflow

  1. git status – see what’s going on
  2. git add . – stage changes
  3. git commit -m "describe the why" – snapshot
  4. git pull – merge remote work first
  5. git push – publish your snapshot

Start a Local Repository

mkdir my-project && cd my-project
git init # creates .git folder
echo "# my-project" > README.md
git add README.md
git commit -m "Initial commit"

Branches Let You Work in Parallel

git branch # list local branches
git switch -c feature # create & checkout
git switch main # jump back
git merge feature # integrate
git branch -d feature # delete if fully merged

Graphical Log

git log --oneline --graph --decorate

Connect to GitHub (or GitLab/Bitbucket)

# on GitHub, create empty repo, then:
git remote add origin https://github.com/YOU/REPO.git
git push -u origin main # first push sets upstream

Clone Existing Project

git clone https://github.com/USER/PROJECT.git

Typical Collaboration Cycle

  1. git pull – grab latest
  2. edit / test
  3. git add -p – stage hunks interactively
  4. git commit
  5. git push

Undo Without Panic

I want to…Command
unstage a filegit restore --staged <file>
discard edits in working filegit restore <file>
change last commit messagegit commit --amend -m "New msg"
revert a whole commitgit revert <hash>
erase last *n* commits (local only)git reset --hard HEAD~n

☠️ Never reset shared history; use revert instead.

Stash – Temporary Shelves

git stash -u # stash tracked + untracked
git stash list
git stash apply stash@{1}
git stash pop # apply & remove top

Clean Untracked Files

git clean -n # dry-run
git clean -fd # force delete

Interactive Rebase (Polish History)

git rebase -i HEAD~4 # squash, re-order, reword

Bisect – Find Bug in O(log n)

git bisect start
git bisect bad # current commit is broken
git bisect good v2.0 # last known good tag
# Git checks out middle; test & mark:
git bisect good|bad
# When done:
git bisect reset

Aliases (Save Keystrokes)

git config --global alias.lg "log --oneline --graph --decorate"
git config --global alias.st "status -sb"
# now use: git lg or git st
